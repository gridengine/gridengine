About the project approval process

When a project is proposed on this site, it appears in the Proposed Projects link in the "Pending Actions" section of the Domain Administrator's Start Page. The next time you log in, this count indicates how many projects are awaiting your action. Taking no action has the same result as deferring a project; they remain effectively in limbo as pending actions.

Click the Proposed Projects link in the "Pending Actions" section to display the Project Approval page. An alphabetical listing of all pending projects appears with the proposer's name and a project summary included. Project titles link to each project's home page. The proposer's name links to the Edit User screen containing this user's account information.

As the Domain Administrator, you disperse with pending projects in one of three ways: approve, defer, or disapprove them by placing checkmarks in the appropriate boxes. All of these actions are tracked in an administrative audit log, along with other administrative actions. See Using administrative audit logs for further information.

The criteria for approving or disapproving proposed projects falls within your judgment as the Domain Administrator. Factors to consider might include:

  • Does the project fall within your site's overall mission and general goals?
  • Is the project well scoped out?
  • Are project objectives clearly stated?

When you approve a project, an email message is automatically generated notifying the Project Owner. The project appears in the site's hosted project listings, including the Hosted Projects page and the appropriate category home page(s). The Project Owner can now approve members and grant roles to them.

Deferring or disapproving projects

While your approval of a project is pending, or when you defer approval, the project simply remains in your list of pending projects awaiting approval. It does not appear within any project listings in public pages or to any other users. The proposer, however, continues to see a link to this project in her/his Start Page under the "My Projects" section, and can conduct most project related activities including:

  • accessing and updating their project's home page
  • inviting participants
  • uploading and downloading project files and documents
  • adding and editing news items
  • checking source code into the project's CVS repository
  • assigning and tracking project issues.
None of this project information on pending projects is visible to other site users, however. And until you approve a pending project, the Project Owner cannot approve project members or grant roles to them.

When you disapprove a project, an email message is automatically generated to notify the proposer. This message is brief; the proposer is encouraged to contact you for further discussion about the specific reasons for disapproval.

Locking, unlocking, and deleting projects

The Project Lock feature enables you to stop all activity on a project if circumstances arise to necessitate this. Whether this is due to site policy matters, technical issues, or other reasons, project locking lets you essentially turn off all site-related project activity. The locked project's home page and other pages can still be accessed and all existing project data is intact, however the Project Owner and members have read-only access to the source code repository, email archives, issue database, project files, and documents. No one -- including the project owner -- can commit source code, enter issues, upload or download files and documents, change web content, or subscribe to or modify mailing lists. If the locked project is a public project, other site users can still view the project's home page but the project is flagged as "locked."

To lock a currently active project, navigate to the project's home page and click on the Admin link in the Project Resources tool bar to display the Project Edit page. Use the Project Lock link at the bottom to display a screen where you can set the lock. This action automatically generates an email notice to the project owner, telling him/her to contact a site administrator immediately.

To unlock a project, your view of the locked project's home page displays the red "Locked" flag and an Unlock link next to it you can use to enable the project again. Alternatively, you can use the Project Edit page where there an Unlock Project link appears at the bottom of the page. When you unlock a project to reactivate it, another email notice is automatically sent to the project owner.

Projects may also be deleted from this site when they either don't pan out or reach the end of their development life cycle. The Project Edit page includes a Project Delete link at the bottom of the page. If you click this link, you are asked to confirm the action. Use the project delete feature with caution! Deleting the project not only removes it from site project listings, but also completely erases the project's cvs repository files, all project files and documents, issues database, and mailing list archives.
